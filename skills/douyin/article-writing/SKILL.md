# Article Writing (长文写作)

## Overview

Douyin's long-form article feature (长图文), launched December 2025, supports up to 8000 characters with 30 images on PC. It displays as auto-scrolling text with background music in the recommendation feed, bridging the gap between short video consumption and deep reading.

## When to Use

**Use when**: Creating long-form articles (300-8000 chars), writing in-depth tutorials, compiling curated lists, creating guides that need detailed explanation.
**Do NOT use when**: Short image-text posts (use image-text-creation), short tips (use short-video-creation).

## Douyin Article Specs

| Spec | Value |
|---|---|
| **Title** | ≤30 characters |
| **Body** | 300-8000 characters |
| **Images** | ≤30 images |
| **Background Music** | Supported |
| **Platform** | PC-only for publishing |

## Core Structure

**Hook (character 1-100)**: Arresting opening — show the value immediately. Readers decide in first 100 chars whether to read.

**Body flow**:
1. Opening: State the problem or promise
2. Content: 3-5 sections, each with clear subheading
3. Visuals: 1 image every 1-2 paragraphs for visual break
4. Summary: Key takeaways at end
5. CTA: What to do next

**Formatting tips**:
- Use short paragraphs (3-5 lines max)
- Add spacing between sections
- Images break text walls
- Bold key points when possible
- Emoji sparingly for visual anchors

## Implementation

1. **Outline first**: 3-5 section structure before writing
2. **Start strong**: First 100 chars = your hook. Promise value.
3. **Section pacing**: Each section ≤500 chars, 1 image
4. **End with CTA**: "关注看更多深度内容" / "收藏慢慢看"
5. **Music**: Select background music that matches content mood

## Common Mistakes

| Mistake | Fix |
|---|---|
| Wall of text, no images | Add image every 1-2 paragraphs |
| Title too long (>30) | Cut to core hook |
| Ignoring recommended feed display | Article shows as scrolling text — front-load value |
| No sections, continuous text | Break into 3-5 clear sections |
