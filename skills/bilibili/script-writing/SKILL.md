# Script Writing (脚本撰写)

## Overview

Script writing for Bilibili bridges the gap between loose outlines and polished presentation. B站 videos require structure — not a word-for-word teleprompter read, but a clear framework that ensures informational value flows logically and keeps viewers engaged through longer runtime.

## Script Types

| Type | Length | Structure |
|---|---|---|
| **Tutorial** | 5-15 min | Problem→Background→Steps→Result→Summary |
| **Review** | 5-10 min | Context→Features→Testing→Pros/Cons→Verdict |
| **Analysis/Commentary** | 8-20 min | Hook→Context→Deep Dive→Opinion→Discussion |
| **Vlog** | 3-8 min | Loose narrative, key moments, emotional arc |
| **List/Ranking** | 5-10 min | Hook→Criteria→Item 1,2,3...→Summary |

## Script Structure

### Act 1: Hook (0-30s)

```
STATE THE VALUE IMMEDIATELY. Not "Hello everyone welcome to my channel."
Wrong: "大家好,欢迎来到我的频道,今天我们来聊聊XX"
Right: "如果你也遇到过XX问题,这个视频会彻底改变你的解决方式"
```

**Hook formulas**:
- "你有没有想过为什么XX?"
- "用了XX年,我终于明白了一个道理"
- "今天这个方法,可能颠覆你对XX的认知"
- "我花了XX小时/XX元测试,结论出乎意料"

### Act 2: Content Body

**For Tutorials**:
1. Background: Why this matters (1-2 min)
2. Prerequisites: What you need (30s)
3. Step 1, 2, 3: Each with demonstration (2-3 min each)
4. Common pitfalls: What can go wrong (1-2 min)
5. Result: Show final outcome (30s-1 min)

**For Reviews**:
1. Context: Why review this product (1 min)
2. Unboxing/First Impressions (1-2 min)
3. Feature breakdown (2-3 min)
4. Real usage test (2-3 min)
5. Pros and Cons (1-2 min)
6. Verdict: Who should buy/not buy (1 min)

**For Analysis/Commentary**:
1. Hook: The provocative question/claim (30s)
2. Context: Background info viewers need (1-2 min)
3. Deep Dive: Main argument with evidence (3-5 min)
4. Counterpoints: Address opposing views (1-2 min)
5. Conclusion: Your final take (1 min)

### Act 3: Outro and CTA

```
1. Summarize: 3 key takeaways (30s)
2. Next topic teaser: "下期我们聊XX" (15s)
3. CTA: 一键三连, comment topic, follow for more (15s)
4. Sign-off: brief, not drawn out
```

## Writing Tips

- Write conversationally — speak the script out loud when drafting
- Include B-roll notes: "[INSERT: product close-up]" / "[SCREEN RECORD: settings page]"
- Mark emphasis points: where to raise voice, pause, speed up
- Time each section — aim for 150-180 chars per minute spoken
- Leave room for improvisation — script is a guide, not a cage

## Templates

### Tutorial Script Template
```
[Hook: 30s] State problem + promise solution
[Background: 1-2min] Why this matters, what you'll learn
[Prep: 30s] What you need before starting
[Step 1: 2-3min] Clear demo + explanation
[Step 2: 2-3min] Clear demo + explanation
[Step 3: 2-3min] Clear demo + explanation
[Pitfalls: 1-2min] What to watch out for
[Result: 30s] Final outcome shown
[Summary: 30s] Key takeaways
[CTA: 15s] 一键三连 + comment + follow
```

### Review Script Template
```
[Hook: 30s] The claim about this product
[Context: 1min] Why I bought/am reviewing this
[Unboxing: 1-2min] First impressions
[Features: 2-3min] Key features, what matters
[Testing: 2-3min] Real usage, real results
[Pros/Cons: 1-2min] Honest breakdown
[Verdict: 1min] Buy or skip, who it's for
[CTA: 15s] 一键三连 + comment your experience
```

## Common Mistakes

| Mistake | Fix |
|---|---|
| Script reads like essay, not speech | Speak it out loud, rewrite unnatural parts |
| Too dense, no B-roll breaks | Add B-roll notes every 1-2 minutes |
| No timing estimates | Time each section, total should hit target |
| Generic CTA | Make CTA specific: "评论区告诉我你用过哪个" |
| No emphasis markers | Mark where to slow down, speed up, pause |
