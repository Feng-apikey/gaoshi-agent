# Article Writing (专栏写作)

## Overview

Bilibili's column (专栏) system supports long-form written content with Markdown formatting, images, and embeds. Unlike short-form platforms, B站 columns allow deep exploration of topics. The audience reads for depth and knowledge — they expect substance, not skimmable listicles.

## B站 Column Specs

| Spec | Value |
|---|---|
| **Title** | ≤80 characters |
| **Body** | No strict limit, 2000-10000+ characters typical |
| **Format** | Markdown supported |
| **Images** | Embedded within text |
| **Videos** | Can embed B站 videos |
| **Cover** | 16:9 horizontal cover image |

## Core Structure

```
COVER IMAGE (16:9, eye-catching, title text overlay)
↓
TITLE (≤80 chars, value + curiosity)
↓
OPENING (100-200 chars): Hook — why read this?
↓
BODY:
  Section 1: Context/Background
  Section 2: Main Content/Argument
  Section 3: Deep Dive/Examples
  Section 4: Alternative Views/Limitations
↓
CONCLUSION: Key takeaways + CTA
```

## Writing Guidelines

### Tone and Voice
- **有态度**: Have a clear viewpoint. B站 readers value authenticity over neutrality.
- **有深度**: Go deeper than short-form content. Provide evidence, examples, data.
- **有幽默**: Wit and self-awareness resonate with B站 community.
- **不标题党**: No clickbait. Title must accurately reflect content.

### Formatting
- Use headers (##, ###) to structure sections
- Bold key points
- Use lists for clarity
- Insert images every 300-500 words
- Embed relevant B站 videos
- Add horizontal rules between major sections

### Pacing
- 2000-5000 words for standard columns
- Each section 3-5 paragraphs
- Vary paragraph length (2-5 sentences)
- Use one-line paragraphs for impact

## Implementation

1. **Outline**: 3-5 main sections with key points
2. **Research**: Gather data, examples, references
3. **Write**: Draft full body, then edit for clarity
4. **Visuals**: Select/design images for each section
5. **Cover**: Design eye-catching 16:9 cover with title
6. **Edit**: Cut 20% — every paragraph earns its place
7. **Publish**: Select correct 分区, add relevant tags

## CTA Examples

- "觉得有用的话,点个赞支持一下"
- "欢迎在评论区讨论你的看法"
- "收藏起来慢慢看,下次找得到"
- "关注我看更多深度内容"
- "一键三连是我更新的最大动力"

## Common Mistakes

| Mistake | Fix |
|---|---|
| Clickbait title, shallow content | Title accurately represents depth |
| Wall of text, no images | Add image every 300-500 words |
| No clear structure | Use headers and sections |
| Too short (<1000 words) | B站 readers expect depth — go deeper |
| Ignoring Markdown | Format properly for readability |
