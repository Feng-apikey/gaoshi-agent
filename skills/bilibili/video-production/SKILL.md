# Video Production (视频制作)

## Overview

Bilibili video production differs fundamentally from Douyin — it's horizontal (16:9), longer (3-15 minutes is standard), deeper in content, and rewards informational value over entertainment speed. B站 viewers actively seek knowledge, skills, and genuine expertise, not just quick dopamine hits. The community values authenticity, depth, and personality over polish.

**Key insight**: B站 algorithm weights watch time AND interaction (弹幕/评论/收藏/投币). Videos that spark discussion and make viewers feel they learned something valuable outperform flashy but shallow content.

## When to Use

**Use when**: Creating horizontal video for B站, making tutorials/how-tos, filming product reviews, vlogging with substance, producing educational series.
**Do NOT use when**: Short vertical clips for Douyin (different format), pure entertainment without depth.

## B站 Specs

| Spec | Value |
|---|---|
| **Aspect Ratio** | 16:9 horizontal |
| **Resolution** | 1920×1080 minimum |
| **Frame Rate** | 30fps (60fps for gaming/content) |
| **Length** | 3-15 minutes (tutorials can be longer) |
| **Format** | MP4 |
| **File Size** | Under 10GB |

## Core Structure

**Intro (0-30s)**: Hook + preview of what you'll learn. No "大家好欢迎来到我的频道" — state the value immediately.

**Body (30s to 80% of video)**:
1. Context: Why this matters
2. Content: Step by step, clear sections
3. Examples: Real demonstrations
4. Depth: Go deeper than short video platforms would allow

**Outro (last 20%)**: Summary + next steps + CTA (一键三连 - like, coin, collect)

## Quick Reference

| Video Type | Optimal Length | Structure |
|---|---|---|
| **Tutorial/教程** | 5-15 min | Hook→Background→Steps→Result→Summary |
| **Review/测评** | 5-10 min | Unboxing→Features→Testing→Pros/Cons→Verdict |
| **Analysis/杂谈** | 8-20 min | Hook→Context→Deep Dive→Opinion→Discussion |
| **Vlog/日常** | 3-8 min | Scene flow, natural narrative |
| **Comparison/对比** | 5-10 min | Criteria→A vs B→Winner→Why |

## Implementation

### 1. Script and Structure
- Write loose script before filming (not word-for-word, but key points)
- Use chapters/sections (B站 supports video chapters)
- Plan for timestamps in description

### 2. Filming
- Horizontal 16:9, stable camera (tripod recommended)
- Good lighting: softbox or natural window light
- External microphone (critical — B站 audience values audio quality)
- B-roll: product shots, screen recordings, demonstration footage

### 3. Editing
- Software: 剪映专业版 / Premiere / DaVinci Resolve
- Keep pacing tighter than you think — cut dead air and "um"s
- Add text overlays for key points (B站 viewers read while watching)
- Chapter markers every 2-5 minutes
- Background music at 10-15% volume
- Clean transitions, no cheesy effects

### 4. Thumbnail/Cover
- 16:9 horizontal cover image
- Face + expressive emotion + bold text
- Text: 3-8 characters, high contrast
- Consistent branding style across videos

### 5. Title and Description

**Title**: ≤80 characters, show value + arouse curiosity
**Description**: Markdown-supported, include:
- Video summary (1-2 paragraphs)
- Timestamps/chapters
- Related links/resources
- CTA: 一键三连, 关注, 评论
- Relevant tags

### 6. Tags and Category
- Select correct 分区 (video zone) — critical for algorithm
- Tags: mix of broad and niche tags
- 热门标签 should be genuinely relevant

## B站-Specific Tips

- Open with value statement, not personal intro
- Respond to comments actively — community building matters
- 弹幕 (danmaku) is both feedback and content — embrace it
- Collaborations (联动) boost growth significantly
- Series/playlists increase binge-watching
- Post consistently — 1-3x per week is sustainable

## Common Mistakes

| Mistake | Fix |
|---|---|
| Slow intro, "Hi everyone" | Start with value/hook immediately |
| Vertical video on B站 | Film horizontal 16:9 |
| No chapters for long video | Add timestamps in description |
| Ignoring audio quality | Invest in mic before camera |
| No CTA or weak CTA | Ask for 一键三连 explicitly |
| Wrong 分区 selection | Choose most accurate category |
